# クイックスタート: 日本語絵文字名対応

**フィーチャーブランチ**: `003-japanese-emoji-name`

## ビルド

```bash
npm run build
```

## 動作確認手順

1. `npm run build` でビルド
2. Chrome で `chrome://extensions/` を開き、`dist/` ディレクトリを読み込む
3. Slackワークスペースの「絵文字カスタマイズ」ページ（`/customize/emoji`）にアクセス
4. 以下のテストファイルをドロップして確認:

### テストケース

| # | ファイル名 | 期待されるリスト表示 | 期待されるAPI結果 |
|---|-----------|-------------------|-----------------|
| 1 | `笑顔.png` | `:笑顔:` | `invalid_name` エラー表示（Slack非対応の場合） |
| 2 | `おはよう.png` | `:おはよう:` | 同上 |
| 3 | `ネコ.png` | `:ネコ:` | 同上 |
| 4 | `Hello_世界.png` | `:hello_世界:` | 同上 |
| 5 | `test.png` | `:test:` | 成功（既存動作） |
| 6 | `My Emoji.png` | `:my_emoji:` | 成功（既存動作） |

### 確認ポイント

- [ ] 日本語ファイル名の絵文字名にひらがな・カタカナ・漢字が保持されている
- [ ] 既存のASCIIファイル名の動作に変更がない
- [ ] APIエラー時に日本語のエラーメッセージが表示される
- [ ] ビルドがエラーなく完了する
